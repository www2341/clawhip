# clawhip build context snapshot

- Task statement: Build clawhip, a standalone Rust event-to-channel notification router that is independent from clawdbot plugins/gateways.
- Desired outcome: Working prototype with CLI event emission, stdin and HTTP ingestion, route/config handling, Discord REST API delivery, and at least end-to-end working `custom` command.
- Known facts/evidence: Fresh repo with only Cargo.toml and src/main.rs. Required deps: reqwest, serde, serde_json, toml, clap, tokio, axum. Config path should be ~/.clawhip/config.toml.
- Constraints: Independent bot using its own Discord bot token. No clawdbot gateway integration. Push to main when done.
- Unknowns/open questions: Exact interactive behavior for `config` command can be minimal prototype if functional. Need decide route matching model and message formatting schema.
- Likely codebase touchpoints: Cargo.toml, src/main.rs, new modules under src/, README/docs if useful.
